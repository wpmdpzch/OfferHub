# 📋 OfferHub 项目需求文档 v5.0

> **项目名称**：OfferHub
> **定位**：开源面试信息聚合平台（内容平台 + 学习工具）
> **目标**：打造万星开源项目，成为开发者面试备战首选平台
> **更新时间**：2026-03-17（v5.1，基于 design_feedback.md 决策更新）

---

## 第一部分：项目愿景与定位 🎯

### 1.1 核心愿景

> "让每个开发者都能拿到心仪 Offer"

**两阶段定位：**

| 阶段 | 定位 | 核心目标 |
|------|------|----------|
| **MVP 阶段** | 信息聚合平台 | 聚合大量有效面试信息，建立内容基础 |
| **成长阶段** | 内容平台 + 学习工具 | 在内容基础上叠加组卷、AI伴读、社区 |

**核心逻辑：先有数据，再做功能。** 没有足够的内容积累，组卷、推荐、AI 都是空中楼阁。

### 1.2 用户痛点

- **信息分散**：面经散落在牛客/知乎/掘金/公众号，题库分布在 GitHub/Gitee/各技术博客
- **信息过载**：搜索结果是海量链接，质量参差不齐，重复内容多
- **时效性差**：面试趋势变化快，旧面经参考价值低，难以追踪最新动态

**OfferHub 解法**：聚合多源信息 → 结构化组织 → 一站式浏览 → 持续更新

## 第二部分：功能优先级 📊

### 2.1 优先级矩阵

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0 MVP** | 信息聚合采集 | GitHub/Gitee API + RSS 自动采集 |
| **P0 MVP** | 内容展示（信息流） | 首页信息流、文章详情、分类/标签浏览 |
| **P0 MVP** | 全文搜索 | PostgreSQL 全文搜索（pg_trgm） |
| **P0 MVP** | 用户系统（基础） | 注册/登录，UGC 投稿入口 |
| **P1** | 内容审核后台 | 管理员审核、采集源管理 |
| **P1** | 互动功能 | 点赞、收藏、评论（不含评论点赞） |
| **P1** | 积分明细 | point_logs 流水表，用户可查看积分来源 |
| **P2** | 题库与组卷 | 结构化题目、智能组卷、在线答题 |
| **P2** | AI 伴读 | 问答、模拟面试、答案评估 |
| **P3** | 社区互动 | 讨论区、私信、动态 |
| **P3** | 企业端 | 岗位发布、人才对接 |
| **P3** | 数据看板 / 插件生态 | 长期规划 |

### 2.2 竞品对比

| 维度 | 牛客/掘金 | JavaGuide/开源题库 | **OfferHub** |
|------|-----------|-------------------|--------------|
| 内容来源 | 自有封闭 | 静态人工整理 | **多源自动聚合** |
| 更新频率 | 人工更新慢 | 几乎不更新 | **自动化持续更新** |
| 搜索能力 | 基础搜索 | 无搜索 | **全文检索** |
| 使用门槛 | 注册/付费 | 免费但无交互 | **开源免费** |
| 学习工具 | 有但付费 | 无 | **后续叠加** |

## 第三部分：MVP 详细设计 🚀

### 3.1 MVP 范围（严格定义）

MVP 只做一件事：**让用户能搜到、看到、找到面试相关信息。**

**包含：**
- 自动采集（GitHub/Gitee API + RSS）
- 内容展示（信息流 + 详情页）
- 分类/标签浏览
- 全文搜索
- 用户注册/登录
- UGC 投稿（基础版，先审后发）

**不包含（MVP 后做）：**
- 点赞/收藏/评论（互动功能整体后置 P1）
- 评论点赞（P2）
- 图片/附件上传（暂不支持）
- 智能组卷
- AI 伴读
- 社区功能
- 企业端

### 3.2 冷启动方案

MVP 上线前的内容积累策略，分三步走：

**Step 1：自动采集种子内容（上线前）**
- 通过 GitHub MCP 工具发现高 Star 面试题仓库（目标 50+ 个）
- 自动采集 Markdown 内容，解析结构化文章
- 目标：上线时已有 500+ 篇内容

**Step 2：RSS 持续补充（上线后自动运行）**
- 订阅掘金、美团技术、阮一峰等技术博客 RSS
- 每日自动入库新文章
- 目标：每日新增 10-20 篇

**Step 3：UGC 激励（有用户后）**
- 开放投稿入口，积分激励体系（注册+10、发文+5、被点赞+1、被收藏+2、每日登录+1、违规-10）
- 积分 > 100 免审，新用户先审后发
- 质量通过用户举报 + 管理员审核双重验证（举报后进入待审核队列，人工处理）
- 不强求初期 UGC 质量，先积累再筛选

### 3.3 内容分类体系

**一级分类（6 类）：**

| 分类 | 子分类示例 |
|------|-----------|
| 面经分享 | 大厂/外企/创业/国企 |
| 技术文章 | 前端/后端/移动端/算法/数据库/云原生/AI |
| 学习路径 | 校招/社招/转行/进阶 |
| 面试技巧 | 简历/准备/谈薪/心态 |
| 行业资讯 | 招聘动态/技术趋势/薪资行情 |
| 题库解析 | 高频考题/难题详解/考点总结 |

### 3.4 MVP 开发排期（重新评估）

基于单人/小团队开发能力，保守估计：

```
Week 1-2: 基础设施
├── 项目脚手架（Next.js + FastAPI）
├── Docker Compose 环境搭建
├── PostgreSQL 数据库初始化（建表/索引）
└── 基础 CI/CD（GitHub Actions）

Week 3-4: 采集系统（核心）
├── GitHub/Gitee API 采集器
├── RSS 采集器
├── 内容解析（Markdown 提取）
├── 去重逻辑（URL 唯一索引 + SimHash 相似度检测）
└── 合规框架（robots.txt / 频率限制）

Week 5-6: 后端 API
├── 文章 CRUD API
├── 分类/标签 API
├── 全文搜索 API（PostgreSQL pg_trgm）
└── 用户注册/登录（JWT）

Week 7-8: 前端展示
├── 信息流首页
├── 文章详情页（Markdown 渲染）
├── 分类/标签浏览页
└── 搜索结果页

Week 9-10: 收尾
├── 管理后台（内容审核 + 采集源管理）
├── UGC 投稿页面
├── SEO 优化（SSR + sitemap）
└── 内测 + Bug 修复
```

**总计：10 周（约 2.5 个月）**，这是保守估计，实际可能更快。

## 第四部分：内容来源与合规 ⚠️

### 4.1 数据源优先级

| 来源 | 合规风险 | 内容质量 | 优先级 |
|------|----------|----------|--------|
| GitHub/Gitee 公开仓库（API） | 极低 | 高（高 Star 仓库） | P0 |
| RSS 订阅（技术博客） | 低（需注明来源） | 中高 | P0 |
| UGC 用户投稿 | 无 | 用户把控 | P1 |
| 搜索引擎公开内容（爬虫） | 中（需谨慎） | 中 | P2 |

### 4.2 合规红线

- 仅采集公开免费内容，不突破登录/付费墙
- 严格遵守 robots.txt
- 请求频率 ≤ 1 req/s，带随机延迟
- User-Agent 标明：`OfferHub-Bot/1.0 (+https://github.com/wpmdpzch/OfferHub)`
- 所有内容标注原始来源链接
- 提供内容删除通道（DMCA 投诉处理）

## 第五部分：开源运营策略 📈

### 5.1 Star 增长路径

| 阶段 | 目标 | 时间 | 关键动作 |
|------|------|------|----------|
| 冷启动 | 0 → 100 Star | 第 1-2 月 | 完善 README、上线 Demo、提交 Awesome 列表 |
| 起步 | 100 → 1K Star | 第 3-6 月 | 掘金/知乎/V2EX 推广、每周迭代 |
| 增长 | 1K → 5K Star | 第 6-12 月 | 技术公众号曝光、高校合作、面经征集活动 |
| 规模 | 5K → 10K+ Star | 第 12-24 月 | 生态建设、企业采用、国际化 |

### 5.2 内容运营

- 上线前：自动采集 500+ 篇种子内容
- 上线后：每日自动更新 10-20 篇（RSS + 定时采集）
- 社区激励：投稿积分体系，优质内容置顶

## 第六部分：风险与应对 ⚠️

| 风险类型 | 具体问题 | 应对策略 | 优先级 |
|----------|----------|----------|--------|
| **法律风险** | 爬虫合规、版权问题 | 优先 API 采集、遵守 robots.txt、DMCA 通道 | 高 |
| **内容风险** | 虚假面经、质量差 | 用户点赞/举报 + 管理员审核双重机制 | 高 |
| **维护风险** | 单人/小团队，作者失去兴趣导致项目荒废 | 早期引入 2-3 名核心贡献者、建立社区治理规范（CONTRIBUTING.md）、关键模块文档化降低接手门槛 | 高 |
| **竞争风险** | 牛客/掘金已有面经板块 | 强调开源免费、聚合优势、可私有部署 | 中 |
| **运营风险** | 内容更新不足、冷启动失败 | 自动化采集兜底、种子内容预积累 | 高 |
| **隐私风险** | 用户信息泄露 | 加密存储、权限控制、最小化数据收集 | 高 |
| **增长风险** | Star 增长缓慢 | 多渠道推广、内容运营、社区建设 | 中 |
