# 📋 OfferHub 项目需求文档 v4.0

> **项目名称**：OfferHub  
> **定位**：开源面试信息聚合器（核心）+ 智能组卷 + AI伴读  
> **目标**：打造万星开源项目，成为开发者面试备战首选平台  
> **更新时间**：2026-03-17

---

## 第一部分：项目愿景与定位 🎯

### 1.1 核心愿景

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   "让每个开发者都能拿到心仪Offer"                                │
│                                                                 │
│   OfferHub = 信息聚合器 ⭐ + 智能组卷 + AI伴读 + 社区            │
│                                                                 │
│   🎯 核心定位：面试信息聚合展示（博客/文章形式）                  │
│   🎯 轻量级 · 聚焦核心 · 开源免费                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 产品定位调整

| 维度 | 原定位 | **新定位（v4.0）** |
|------|--------|-------------------|
| **核心功能** | 题库+组卷 | **信息聚合展示（博客/文章）** |
| **优先级** | 题库P0 | **信息聚合P0，题库P1** |
| **用户价值** | 刷题自测 | **快速获取面试信息+系统学习** |
| **差异化** | 智能组卷 | **聚合展示+内容组织** |

### 1.3 为什么信息聚合是核心？

```
┌─────────────────────────────────────────────────────────────────┐
│                    用户痛点分析                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  😫 痛点1：信息分散                                             │
│  • 面经散落在牛客/知乎/掘金/公众号                               │
│  • 题库分布在GitHub/Gitee/各技术博客                            │
│  • 学习资源分散，难以系统整理                                   │
│                                                                 │
│  😫 痛点2：信息过载                                             │
│  • 搜索结果是海量链接，需要逐个点击                             │
│  • 质量参差不齐，难以辨别                                       │
│  • 重复内容多，浪费时间                                         │
│                                                                 │
│  😫 痛点3：信息时效性                                           │
│  • 面试趋势变化快，旧面经参考价值低                             │
│  • 新技术考点不断出现                                           │
│  • 难以追踪最新面试动态                                         │
│                                                                 │
│  ✅ OfferHub解决方案：                                           │
│  • 聚合多源信息，一站式浏览                                     │
│  • 结构化组织，博客/文章形式展示                                │
│  • 智能分类检索，快速定位                                       │
│  • 持续更新，保证时效性                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 与竞品对比（突出信息聚合优势）

| 维度 | 商业SaaS | 开源题库 | **OfferHub** |
|------|----------|----------|--------------|
| 信息聚合 | 自有内容 | 静态整理 | **多源聚合+持续更新** |
| 展示形式 | APP/网页 | README/Markdown | **博客/文章流** |
| 内容组织 | 分类有限 | 简单分类 | **智能分类+标签体系** |

| 用户贡献 | 封闭 | 需PR | **便捷投稿+审核** |
